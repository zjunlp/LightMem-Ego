# LightMem Glass

LightMem Glass is an Android app for Rokid AI Glass. It captures the glass camera and microphone in real time, streams them to a LightMem-compatible backend, and lets the user ask short voice questions from the glasses UI.

The app is based on [Rokid's bare-device Android sample style](https://custom.rokid.com/prod/rokid_web/ff28c865a9634876be98cbc293588460/pc/cn/index.html?documentId=201f7e36b17b4a4389ea4c38a4f23381): standard Android APIs, Jetpack Compose UI, CameraX camera capture, `AudioRecord` microphone capture, and touchpad / glass-button input. It does not require a phone-side SDK at runtime.

## What This App Does

- Starts a real-time capture session with a backend API.
- Sends camera frames and microphone audio to the backend.
- Uses RTMP live push when the backend returns a `push_url`.
- Falls back to HTTP frame/audio upload when RTMP setup fails.
- Shows answers on a 480 x 640 glasses display using a high-contrast green-on-black UI.
- Supports voice questions from the glasses.

This open-source version does not include local session recording, replay-from-file mode, preset-question UI, or standalone sample screens for photo/video/audio/IMU testing.

## Terms

- API: the server address and endpoints that the app talks to. For example, the app starts a session by calling the backend API.
- APK: the Android install package generated by Gradle.
- ADB: Android Debug Bridge, the command-line tool used to install and debug apps on Android devices.
- RTMP: a live video streaming protocol. In this app it is used to push the camera stream to the backend when available.
- Gradle wrapper: the included `gradlew` / `gradlew.bat` scripts that build the Android project without requiring a manually installed Gradle version.
- Jetpack Compose: Android's modern UI toolkit. The app uses it to draw the glasses screen in Kotlin code.
- CameraX: Android's camera library. The app uses it to read camera frames for backend upload.

## Requirements

- Rokid AI Glass running YodaOS-Sprite / Android 12 API 31 or later.
- Android Studio or Android SDK command-line tools.
- JDK compatible with the Android Gradle Plugin used by this project.
- A backend implementing the LightMem / Rokid stream API.
- ADB access to the glasses. Rokid's bare-device docs describe using a dedicated development cable and enabling glasses ADB from the Rokid AI mobile app.

The project currently uses:

- `minSdk = 31`
- `targetSdk = 36`
- package name / application id: `cn.zjukg.lightmem.glass`

## Configure The Backend

Edit:

```text
app/src/main/java/cn/zjukg/lightmem/glass/worldmm/WorldMMConfig.kt
```

Important values:

```kotlin
const val API_BASE_URL = "https://lightmem-ego.zjukg.cn/api"
const val INPUT_MODE = "rokid_live_rtmp"
const val FALLBACK_INPUT_MODE = "rokid_frame_audio"
```

For public use, change `API_BASE_URL` to your own backend before building. If you leave the default value, the app will try to connect to the configured LightMem server.

## Install And Start On Rokid AI Glass

1. Enable ADB for the glasses using the Rokid AI mobile app and the dedicated development cable.
2. Check that the device is visible:

```bash
adb devices
```

3. Install the APK:

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

4. Start the app from the glasses launcher, or start it with ADB:

```bash
adb shell monkey -p cn.zjukg.lightmem.glass 1
```

5. Watch logs if needed:

```bash
adb logcat | grep OmniSparkDiag
```

On Windows PowerShell, use `findstr` instead of `grep`:

```powershell
adb logcat | findstr OmniSparkDiag
```

## Supported Operations

The app follows the Rokid bare-device input model described in `rokid_docs/`.

- LightMem screen:
  - Long press: start or stop the real-time capture session. A capture session sends camera frames and microphone audio to the configured backend.
  - Sprite click while a session is running: start recording a voice question. Click it again to stop recording and submit the question to the backend.
  - One-finger click / Enter key: show the next answer page when a backend answer has multiple pages.
  - Two-finger double click: show the previous answer page when a backend answer has multiple pages.
  - Back / one-finger double click: consumed by the app to avoid accidental exit during glasses use.

There is no preset-question list in the current open-source build. All user questions are captured from the microphone.

The exact key mapping is implemented in:

```text
app/src/main/java/cn/zjukg/lightmem/glass/input/
```

## Permissions

The app only declares permissions needed for the real-time LightMem flow:

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.RECORD_AUDIO" />
```

- `CAMERA`: captures frames from the glasses camera.
- `RECORD_AUDIO`: captures microphone audio and voice questions.
- `INTERNET`: talks to the backend API and streams data.

No external-storage permission is required in the current version because local recording and replay-from-file mode have been removed.

Android automatic backup is disabled with `android:allowBackup="false"` so local app state is not copied into device/cloud backup by default.

## Privacy Notes

This app captures camera frames and microphone audio while a session is running and sends them to the configured backend. It does not save local session recordings or expose standalone local recording screens in the current open-source version.

Before using the app with real users, make sure the backend URL, data retention policy, and user consent flow match your deployment requirements.

## Development Notes

- UI should stay within the Rokid glasses safe display area. The project uses compact green-on-black Compose components following the 480 x 640 glasses UI constraints in the Rokid UI design notes under `rokid_docs/`.
- Camera capture avoids a full preview and uses CameraX analysis frames for upload.
- Microphone capture uses `AudioRecord` and downmixes Rokid's multi-channel input before upload.
- RTMP streaming is handled by RootEncoder when the backend returns a `push_url`.
- Generated build output, IDE state, and `local.properties` should stay out of Git. The included `.gitignore` already excludes these local files.
